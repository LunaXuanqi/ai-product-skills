# 实习经历沉淀 Skill

将你实习期间散落在各处的工作记录，整理为**可直接粘进简历的产品经理项目经历**。

---

## 适用场景

- 实习快结束了，需要把做过的事情整理成简历项目经历
- 日常积累：定期把工作记录沉淀下来，避免最后想不起来做了什么
- 投不同公司需要不同版本的简历

---

## 输入

可以接受以下任意形式的原始素材：

- 飞书文档 / 待办事项 / 每日记录
- 工作方案、调研报告、PRD
- 会议纪要、聊天记录摘要
- OKR 及进度记录
- 任何你觉得"做过但还没整理"的内容

**直接把原始内容丢给我，不需要预处理。越乱越杂越好，整理是我的事。**

---

## 处理流程

### Step 1：提取事实

从原始素材中提取：

| 维度 | 要提取的信息 |
|------|-------------|
| **做了什么** | 具体负责的功能/项目/模块 |
| **背景和目标** | 为什么要做这件事，解决什么问题 |
| **方法和过程** | 怎么做的（调研、分析、设计、推动） |
| **产出物** | 交付了什么（方案文档、功能上线、数据看板等） |
| **量化结果** | 能量化的数字（覆盖 X 个功能、提升 X%、X 条用户反馈） |
| **协作关系** | 和谁合作、你在其中的角色 |

### Step 2：结构化梳理

将提取的事实按**项目维度**归类整理，每个项目输出：

```
## 项目名称：[一句话概括]

- **背景**：为什么做这件事
- **你的角色**：你具体负责什么（强调独立负责 > 参与）
- **做了什么**：
  1. [具体动作 1]
  2. [具体动作 2]
  3. [具体动作 3]
- **方法论**：用了什么方法（调研框架、分析模型、设计流程等）
- **产出**：交付了什么
- **量化结果**：可衡量的成果
- **亮点**：最能体现你能力的一个点
```

### Step 3：提炼卖点

从结构化梳理中，为每个项目提炼 **2-3 个最能打的卖点**。

筛选标准：
- **优先选有量化结果的**：有数字 > 没数字
- **优先选你主导的**：独立负责 > 配合参与
- **优先选有方法论的**：体现思考过程 > 只说做了什么
- **优先选和目标岗位相关的**：匹配 JD > 泛泛而谈

每个项目最终只保留最能打的点写进简历，其余留在结构化梳理里备用（面试时可以展开讲）。

### Step 4：输出简历文本

为每个项目输出 **三个版本**，适配不同场景：

#### 一句话版（用于简历空间极紧张时）

```
负责海外版 10 个 Native 特色功能从调研到落地，推动 3 个功能上线并获得用户正向反馈
```

#### 标准版（3-4 行 bullet points，最常用）

```
- 独立负责码上飞海外版特色功能设计，通过 5 个竞品深度调研 + 海外用户访谈，识别 X 个核心差异化需求
- 基于调研洞察设计 10 个 Native 特色功能方案，其中 3 个推动上线，获得 Discord 社区 20+ 条正向反馈
- 建立海外功能优先级评估框架，沉淀竞品分析模板和用户场景地图，被团队持续复用
```

#### 详细版（5-6 行，用于重点突出的项目）

在标准版基础上展开方法论和具体案例。

**格式规范：**
- 每条 bullet 以**动词开头**（负责、主导、设计、推动、建立、完成、输出）
- 结构为**动作 + 方法 + 结果**，一条 bullet 内形成闭环
- 数字具体化（"X 个竞品"而不是"多个竞品"，"X 条反馈"而不是"好评"）
- 不用 STAR 标签（S/T/A/R），直接用自然语言

### Step 5：中英文双版本

每个版本同时输出中文和英文。

英文版注意：
- 动词用过去时（Led, Designed, Conducted, Delivered）
- 开头动词要有力（Led > Helped, Designed > Worked on, Drove > Participated in）
- 数字写法统一（10+, 3x, 20%）
- 不是逐字翻译，而是按英文简历习惯重写

---

## 输出质量标准

### 产品经理面试官视角的三层检查

**第一层：分量感**
- [ ] 描述体现的是"独立负责"而不是"参与"
- [ ] 项目范围清晰、有明确边界
- [ ] 看起来是一个完整的事，不是打杂的集合

**第二层：专业度**
- [ ] 能看出决策逻辑（为什么做 A 不做 B）
- [ ] 有方法论支撑（调研 → 分析 → 方案 → 验证）
- [ ] 体现产品思维，不只是执行

**第三层：结果可信度**
- [ ] 有具体数字或可量化的产出
- [ ] 数字合理，不夸大（实习生不需要"DAU 提升 50%"）
- [ ] 以下几种结果都可以接受且有说服力：
  - 功能上线 + 用户反馈
  - 方案被采纳并进入开发
  - 调研产出 + 框架沉淀
  - 流程优化 + 效率提升

### 避免的写法

| 弱写法 | 强写法 |
|--------|--------|
| 参与了海外功能讨论 | 独立负责海外版 X 个特色功能设计 |
| 做了用户调研 | 通过竞品调研（X 个产品）+ 用户访谈，识别 X 个核心需求 |
| 帮助提升了用户体验 | 推动 X 个功能上线，获得 Discord 社区 X 条正向反馈 |
| 写了 PRD | 输出 PRD 并推动跨团队落地，从方案到上线周期 X 天 |

## 使用方式

### 日常积累（建议每 1-2 周一次）

```
这是我最近两周的工作记录：[粘贴原始内容]
请帮我提取关键事实，归类到对应的项目下。
```

### 阶段性整理

```
这是我实习期间所有的工作记录和文档：[粘贴/提供文件]
请按处理流程帮我完整整理，输出结构化梳理 + 标准版简历文本（中英文）。
```

### 针对特定岗位定制

```
我要投 [XX 公司] 的 [XX 岗位]，JD 如下：[粘贴 JD]
请基于我的项目经历，挑选最相关的卖点，输出匹配这个岗位的简历文本。
```
