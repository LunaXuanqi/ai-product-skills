# 面试项目介绍 & 模拟考核 Skill

基于 `intern-experience` 整理好的项目经历，生成**面试时口头介绍的话术**和**高频考核问答**。

---

## 前置依赖

先用 `cv-skill` 整理好结构化项目经历和简历文本，再用本 skill 生成面试话术。

---

## 输入

- `cv-skill` 输出的结构化项目梳理
- 或者已经写好的简历项目经历文本

---

## 输出

每个项目输出三部分：

### Part 1：口头介绍话术（2-3 分钟，STAR 框架）

面试官问"介绍一下你做的这个项目"时，你直接说的内容。用 STAR 框架组织叙事结构：

**话术结构：**

```
【S - Situation】背景是什么
  公司/产品是什么，面对什么市场环境或业务挑战（2-3 句话，简洁交代）

【T - Task】你的任务是什么
  你被赋予或主动承担了什么具体职责（1-2 句话，强调独立负责）

【A - Action】你怎么做的（重点，占 40-50% 篇幅）
  关键决策和方法，体现产品思维：
  - 你怎么分析问题的（调研、访谈、数据）
  - 你做了什么取舍（为什么做 A 不做 B）
  - 你怎么推动落地的（跨团队协作、排优先级）

【R - Result】结果如何
  量化产出 + 定性反馈（2-3 句话收尾）
```

**话术标准：**

- 控制在 2-3 分钟，不超过 400 字
- 用口语化表达，不是念简历
- 重点放在"怎么做的"（占 40-50% 篇幅），而不是背景铺垫
- 必须体现产品思维，具体包括：
  - **用户视角**：从用户需求出发，不是从功能出发
  - **取舍判断**：为什么做 A 不做 B
  - **数据意识**：用什么指标衡量成功
  - **闭环思维**：从发现问题到解决问题到验证效果的完整链路

### Part 2：模拟考核问题（5-8 个）

面试官针对这个项目可能追问的问题，按类型分类：

| 类型 | 考察点 | 示例问题 |
|------|--------|----------|
| **需求判断** | 你怎么决定做什么 | "这个需求是怎么来的？你怎么判断优先级？" |
| **用户理解** | 你有多了解用户 | "目标用户是谁？他们最大的痛点是什么？" |
| **方案设计** | 你的产品能力 | "为什么选择这个方案？考虑过哪些替代方案？" |
| **数据思维** | 你怎么衡量效果 | "你怎么定义这个功能的成功指标？" |
| **取舍与挑战** | 你的决策质量 | "过程中遇到最大的挑战是什么？你怎么解决的？" |
| **协作沟通** | 你的推动力 | "你怎么和开发/运营协作推动这件事？" |
| **反思复盘** | 你的成长性 | "如果重新做，你会有什么不一样的做法？" |

每个项目根据实际内容，从以上类型中挑选 5-8 个最可能被问到的问题。

### Part 3：参考回答

每个考核问题给出参考回答，遵循以下原则：

**回答结构：**
```
【结论先行】先给一句话直接回答
【展开逻辑】用 1-2 个层次展开（原因 / 过程 / 对比）
【落地举例】用项目中的具体例子支撑
【收尾】回扣结论或延伸一句思考
```

**回答标准：**

- 每个回答控制在 30 秒 - 1 分钟（150-250 字）
- 不说空话套话（"我觉得用户体验很重要"）
- 要有具体细节（"我调研了 5 个竞品后发现..."）
- 诚实面对不足（"当时数据基础比较弱，所以我用了定性方法..."比编数据强）
- 实习生不需要假装资深，展示学习能力和思考深度比结果更重要

---

## 产品思维检查清单

生成完话术和问答后，用以下清单自检：

- [ ] 介绍里有没有体现"从用户需求出发"而不是"从老板需求出发"
- [ ] 有没有讲清楚至少一个"为什么做 A 不做 B"的取舍
- [ ] 有没有提到你怎么衡量效果（即使是定性的）
- [ ] 有没有展示完整的思考链路（发现问题 → 分析 → 方案 → 验证）
- [ ] 回答里有没有具体的数字或例子支撑，而不是纯抽象描述
- [ ] 面对"不足"类问题，回答是否诚实且有反思

---

## 使用方式

### 基于已有项目经历生成

```
这是我用 intern-experience 整理好的项目经历：[粘贴内容]
请帮我生成每个项目的口头介绍话术 + 模拟考核问题 + 参考回答。
```

### 针对特定公司/岗位定制

```
我要面试 [XX 公司] 的 [XX 岗位]，JD 如下：[粘贴 JD]
这是我的项目经历：[粘贴内容]
请针对这个岗位的要求，调整介绍重点和考核问题。
```

### 模拟面试练习

```
请扮演 [XX 公司] 的产品经理面试官，基于我的项目经历向我提问。
我回答之后，请给我反馈：哪里答得好、哪里可以改进。
```
