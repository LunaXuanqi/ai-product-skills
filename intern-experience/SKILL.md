# 实习经历沉淀 Skill

将你实习期间散落在各处的工作记录，整理为**可写进简历的产品经理项目经历**。

---

## 适用场景

- 实习快结束了，需要把做过的事情整理成简历项目经历
- 日常积累：定期把工作记录沉淀下来，避免最后想不起来做了什么
- 准备面试：把项目经历转化为面试可讲的 STAR 故事

---

## 输入

可以接受以下任意形式的原始素材：

- 飞书文档 / 待办事项 / 每日记录
- 工作方案、调研报告、PRD
- 会议纪要、聊天记录摘要
- OKR 及进度记录
- 任何你觉得"做过但还没整理"的内容

**直接把原始内容丢给我，不需要预处理。越乱越杂越好，整理是我的事。**

---

## 处理流程

### Step 1：提取事实

从原始素材中提取：

| 维度 | 要提取的信息 |
|------|-------------|
| **做了什么** | 具体负责的功能/项目/模块 |
| **背景和目标** | 为什么要做这件事，解决什么问题 |
| **方法和过程** | 怎么做的（调研、分析、设计、推动） |
| **产出物** | 交付了什么（方案文档、功能上线、数据看板等） |
| **量化结果** | 能量化的数字（覆盖 X 个功能、提升 X%、X 条用户反馈） |
| **协作关系** | 和谁合作、你在其中的角色 |

### Step 2：结构化梳理

将提取的事实按**项目维度**归类整理，每个项目输出：

```
## 项目名称：[一句话概括]

- **背景**：为什么做这件事
- **你的角色**：你具体负责什么（强调独立负责 > 参与）
- **做了什么**：
  1. [具体动作 1]
  2. [具体动作 2]
  3. [具体动作 3]
- **方法论**：用了什么方法（调研框架、分析模型、设计流程等）
- **产出**：交付了什么
- **量化结果**：可衡量的成果
- **亮点**：最能体现你能力的一个点
```

### Step 3：输出 STAR 格式简历文本

将每个项目转化为简历可用的 STAR 描述：

```
**S（Situation）**：在什么公司/产品，面对什么背景和挑战
**T（Task）**：你被赋予/主动承担了什么任务
**A（Action）**：你具体采取了哪些行动（方法 + 过程）
**R（Result）**：产出了什么结果（量化优先）
```

最终输出一段 **3-5 行的简历项目描述**，可以直接粘贴到简历中。

---

## 输出质量标准

### 产品经理面试官视角的三层检查

**第一层：分量感**
- [ ] 描述体现的是"独立负责"而不是"参与"
- [ ] 项目范围清晰、有明确边界
- [ ] 看起来是一个完整的事，不是打杂的集合

**第二层：专业度**
- [ ] 能看出决策逻辑（为什么做 A 不做 B）
- [ ] 有方法论支撑（调研 → 分析 → 方案 → 验证）
- [ ] 体现产品思维，不只是执行

**第三层：结果可信度**
- [ ] 有具体数字或可量化的产出
- [ ] 数字合理，不夸大（实习生不需要"DAU 提升 50%"）
- [ ] 以下几种结果都可以接受且有说服力：
  - 功能上线 + 用户反馈
  - 方案被采纳并进入开发
  - 调研产出 + 框架沉淀
  - 流程优化 + 效率提升

### 避免的写法

| 弱写法 | 强写法 |
|--------|--------|
| 参与了海外功能讨论 | 独立负责海外版 X 个特色功能设计 |
| 做了用户调研 | 通过竞品调研（X 个产品）+ 用户访谈，识别 X 个核心需求 |
| 帮助提升了用户体验 | 推动 X 个功能上线，获得 Discord 社区 X 条正向反馈 |
| 写了 PRD | 输出 PRD 并推动跨团队落地，从方案到上线周期 X 天 |

## 使用方式

### 日常积累（建议每 1-2 周一次）

```
这是我最近两周的工作记录：[粘贴原始内容]
请帮我提取关键事实，归类到对应的项目下。
```

### 阶段性整理

```
这是我实习期间所有的工作记录和文档：[粘贴/提供文件]
请按处理流程帮我完整整理，输出结构化梳理 + STAR 格式简历文本。
```

### 面试准备

```
我要面试 [XX 公司] 的产品经理岗位，这是我的项目经历：[粘贴结构化梳理]
请帮我针对这个岗位优化 STAR 描述，突出最相关的能力点。
```
